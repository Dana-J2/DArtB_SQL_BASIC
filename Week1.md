## 데이터 관련 기본 지식  
### [ 데이터 저장 형태 ]  
: 보통 데이터베이스 테이블에 저장  
\* Database(DB) : 데이터의 저장소  
\* Table : 데이터가 저장된 공간  
### [ 데이터 저장 장소 ]  
: <u>*MySQL, Oracle, PostgreSQL*</u> 같은 데이터베이스에 주로 저장 (기업마다 다름)  
=> 특징 : **OLTP**  
### [ OLTP : Online Transaction Processing ]  
: 거래를 하기 위해 사용되는 데이터베이스  
- 보류, 중간 상태가 없음 ( : ***데이터 무결성*** )  
- 데이터 추가(INSERT), 데이터 변경(UPDATE) 빈번하게 발생  
- 분석이 아닌 transaction을 위해 만든 데이터베이스이므로 데이터 추출시 쿼리 속도 느릴 수 있음  
### [ SQL : Structured Query Language ]  
: 데이터베이스에서 데이터를 가지고 올 때, 데이터를 관리할 때 사용하는 언어  
### [ 테이블 형태 ]  
1. **행(ROW)** : 가로로 한줄 / 하나의 row가 하나의 고유한 데이터  
2. **열(Column)** : 세로로 한줄 / 각 데이터의 특정 속성 값  
### [ OLAP & 데이터 웨어하우스 ]  
#### 데이터 웨어하우스(DW)  
: 데이터를 한 곳에 모아서 저장  
- 데이터를 기반한 의사 결정이 가능
- 여러 소스의 데이터를 통합해서 분석이 가능
- 과거 데이터 분석
#### OLAP : Online Analytical Processing  
: 트렌섹션에 집중된 OLTP의 기능을 넘어 대용량 데이터를 분석할 수 있도록  
=> OLTP의 속도, 기능 부족 이슈로 인해 등장  
<br> 

## BigQuery 
### BigQuery란?
: Google Cloud의 OLAP + Data Warehouse  
### BigQuery 장점  
- 기존 SQL문법 통해 사용 가능   
- 속도 빠름  
- Firebase, Google Analytics4 데이터 쉽게 추출 가능  
- 데이터 웨어하우스 사용위해 서버 띄울 필요없음(클라우드 기반)

*"데이터를 추출하기 전에 데이터 웨어하우스에 **데이터가 어떻게 저장**되어있는지 확인하는 과정 필요"*  

## 데이터 저장 형태를 아는 방법 
### ERD : Entity Relationship Diagram
: 데이터베이스 구조를 한눈에 알아보기 위해 사용
- { 테이블 & 컬럼 } 으로 구성된 연결표 형태
- 각 테이블 간의 연결형태 표현됨
### ERD가 없는 경우
: 모든 데이터베이스를 직접 보며 탐색  
- 어떤 { 테이블 & 컬럼 } 이 존재하는가?
- 다른 테이블과 연결할 때 어떤 컬럼을 사용?
- 컬럼 값의 의미?

## 데이터 저장 형태 예시
<U> *Ex1) 서비스에 사용될 DB* </U>  
: 유저 테이블 / 배송 테이블 / 물건 테이블  >>> **"결과"**  
<U> *Ex2) 앱,웹 로그 DB* </U>  
: 회원가입 / 페이지확인 / 컨텐츠확인  >>> **"과정"**

## 문제 2

> **🧚Q. 이번 강의를 통해 SQL이 왜 필요하다고 느끼는지, SQL을 통해 본인이 어떤 것을 해내고 싶은지를 자유롭게 작성해보세요.**

~~~
이번 강의를 통해 SQL은 데이터베이스에 저장된 데이터를 직접 탐색하고 필요한 정보를 추출하기 위해 반드시 필요한 언어라고 느꼈습니다. 
또한 데이터를 활용하는 과정에서 단순히 데이터를 보는 것이 아니라 필요한 데이터를 찾고, 필터링하고, 요약하여 의미 있는 정보를 도출하는 과정이 중요하다는 것을 알게 되었습니다.

앞으로 금융권 진로를 준비하면서 고객 거래 데이터, 소비 패턴 데이터, 금융 상품 이용 데이터 등 다양한 데이터를 분석하게 될 것이라고 생각합니다. 
이때 SQL을 활용하면 데이터베이스에 저장된 대규모 데이터를 효율적으로 분석할 수 있을 것이라고 생각합니다.

따라서 앞으로 SQL 정규과제를 성실히 수행하며 데이터를 직접 탐색하고 추출하는 능력을 길러 이를 기반으로 금융 데이터 분석이나 고객 행동 분석과 같은 데이터 기반 의사결정에 기여할 수 있는 역량을 키우고 싶습니다.
~~~
